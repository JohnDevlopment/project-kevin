Actor

Inherits from: KinematicBody2D

Kinematic Actor type

Description

Extends the functionality of KinematicBody2D with gravity and a vector speed and velocity. By default, the physics process handles gravity with linear interpolation.

In addition to the exported properties, there are some variables that you can access that are not exported to the editor. Here is a list of them:

float gravity_value = 98
The maximum Y velocity the actor can have due to gravity. Defaults to the value of physics/2d/default_gravity in the project settings.

Vector2 velocity = Vector2()
The current velocity of the actor. For the most part, this variable must be manually updated, except that the Y component is affected by gravity every physics step.

Properties

• Vector2 speed_cap — Speed cap of the actor

Default Vector2()

The max speed of the actor. As this property is not acted upon in Actor, it is up to the programmer whether to use it. That being said, it is meant to be used to cap the actor's speed, hence the name.

Methods

• void enable_actor(bool flag)

Enable/disable the Actor

Call this function to enable or disable the actor. Affects the visibility and collision of the actor. The argument flag can be true or false; true to enable the actor's collision and rendering, or false to disable it.

• void enable_collision(bool flag)

Enable/disabled collision

Call this function to enable or disable the actor's collision. The argument flag can be true or false; true to enable the actor's collision, or false to disable it.

• virtual Vector2 get_center() const

Returns the center of the actor

Call this function to return the center of the actor. In its base form, this function returns an empty Vector2, but you can override it to return the actual center of your actor.
