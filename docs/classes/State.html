State

Inherits from: Node

A self-contained state, to be used with StateMachine

Signals

• state_change_request(int new_state)

Indicates that the state should be changed.

The StateMachine reacts to this signal and changes to the state indiciated by new_state.

• state_parent_call(String name, Array args)

A signal that gets communicated to the StateMachine parent and causes it to do something.

The StateMachine parent will connect to this signal automatically. What happens next depends on the name parameter. (See the documentation for StateMachine for details).

Methods

• virtual void cleanup()

Called just before exiting the state.

• Variant get_meta_or_default(String name, Variant default = null) const

Gets a meta value or a default on failure.

If the meta field name exists, that is returned; otherwise default is returned.

• virtual Variant physics_main(float _delta)

Should be called during the physics step.

Override this method to implement custom physics-step behavior.

• virtual Variant process_main(float _delta)

Should be called during the idle step.

Override this method to implement custom idle-step behavior.

• void setup(Variant persistant_state, Variant user_data = null)

Called on the first frame of a state.

Call this function to initialize the state. persistant_state is the persistent state, that is, the object which this state modifies. user_data is forwarded to the state and can be used by it.

Note: if the method _setup is defined, it is called right at the end of this function. So it can be used to implement state-specific initialization.
